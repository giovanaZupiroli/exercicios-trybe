Unix e Bash - parte 1  DIA 1.1
Primeiro nivel, Kernel.
Segundo nivel, SHell. SO > voce> kernel = sh bash... 
terceiro nivel, aplicativos. Cada processo possui um PID. 
/ diretorio raiz - root / boot, bin, dev, etc, home, proc, root, sbin, tmp, usr, var. 
comando ls = lista todos os arquivos e diretórios daquele diretorio. 
ls -a = mostra os mesmos diretorios e mais os arquivos ocultos. 
cd = change directory. 
clear limpa a tela 
mkdir = make directory, criar pasta 
cd. = referencia o proprio diretorio 
cd.. = volta pro diretorio pai, o anterior
pwd = mostra o diretorio completo 

cp exemplo.txt exemplo2.txt = faz uma copia do arquivo no mesmo diretório 
cat exemplo.txt = mostra o conteudo do arquivo 
mv exemplo2.txt /Users/Giovana/Musica = move o arquivo citado para a pasta especificada 
rm exemplo2.txt = remove o arquivo, apaga daquela pasta. 
rmdir pastaexemplo = apaga uma pasta do diretorio, mas a pasta precisa estar vazia, nao pode conter nenhum arquivo 
rm -rf pastaexemplo = apaga tudo, a pasta com os arquivos dentro e tudo mais. 


less = mostra parte do conteúdo por página. A barra de espaço mostra as outras páginas. Apertar Q sai desse modo de exibição. 
head = mostra os 10 primeiros valores/ linhas daquele arquivo. 
head -5 = mostra as 5 primeiras linhas, por exemplo. 
tail = mostra as 10 ultimas linhas/ valores.
tail -5 = mostra os 5 ultimos valores/linhas. 
usando o Less também consigo usar o /exemplo (ao colocar uma barra ele faz a procura desse exemplo. Ai se usa o N para andar pela proxima entrada de busca.) 
grep Toy exemplo.txt = mostra os valores encontrados, são case sensitive. 
grep -toy exemplo.txt = encontra o valor sem ser case sensitive 
grep -i 'o poderoso' exemplo.txt = traz uma pesquisa de frase. 
grep -iv 'o poderoso' exemplo.txt = mostra todos os valores que nao tem essa frase 
grep -in 'o poderoso' exemplo.txt = mostra em qual posição ele esta 
grep -ic 'o poderoso' exemplo.txt = mostra quantas vezes ele foi encontrado no arquivo.
